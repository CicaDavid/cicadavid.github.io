Three Great GitHub Recipes

Make a new local repo & set upstream

  1. on Github, move to source page for repo ("original")
  2. use Fork button to create your Github copy ("new repo")
  3. change new repo Setting to allow Issues
  4. copy the new repo URL
  5. change to local directory where you want to deposit the repo
  6. issue command: git clone <url>
  7. copy URL from original, for upstream
  8. issue command: git remote add upstream <url>
check by issuing command: git remote -v you should see the master point to your new repo and the upstream point to the original.

Fetch upstream changes

  1. shift (use command: git co) to the origin directory
  2. check by issuing command: git remote -v
  3. issue command: git fetch upstream
  4. issue command: git branch -va
  5. (output should include "upstream/master")
  6. shift to master branch
  7. issue command: git merge upstream/master
you may need to commit, or add a commit message (if editor appears, "save") confirm by issuing command: git push origin master

Push task(s) / push branch

  1. issue your choice of commands:
    • git add
    • git commit
    • git commit -a
    to add changes to the head and commit the changes
  2. send to Github; issue command: git push origin <branch>
  3. checkout master
  4. Fast Forward the master; issue command: git merge <branch>
